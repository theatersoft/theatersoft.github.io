bus

Note the distinction between the Node.js runtime and a node element of a bus network.

The message bus is the foundation of the Theatersoft platform. It provides messaging and other features needed to create distributed applications running on Node.js servers and web browser clients.

API

A bus node implements this code:

import {bus} from '@theatersoft/bus'

bus.start(context)

… where context is an object that declares how to create the connection to the message bus. A Node.js node may omit the context and the environment vars BUS and AUTH will be used instead.

type Context

A Context contains a parent and/or a children property: 

export type Context = {
    parent? :Parent,
    children? :Children
}

type Parent = {
    url :string,
    auth :?string
}

type Children = {
    server :any,
    host :string,
    port :number,
    check :(string) => Promise<boolean>
}

Context examples:

{parent: {url: 'ws://localhost:5453'}} creates a connection from a browser to a server at the specified url.

{children: {port: 5453}} creates a new WebSocketServer on port 5453.

{children: {server}} creates a new WebSocketServer and add it to an existing http.Server or https.Server.
